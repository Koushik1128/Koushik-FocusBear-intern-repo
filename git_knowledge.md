# Git Knowledge

**Have you used Git before? If so, in what context?**

Yes, I'd used Git before this internship, mainly in university coursework and project submissions. That was mostly the basics though, clone, add, commit, push on my own branch, without much need to deal with conflicts or collaborating with other people on the same repo at the same time. This internship is really the first time I've used it in a more ongoing, real workflow way, cloning a personal intern repo, working through issues one by one, and pushing changes regularly rather than just submitting something once at the end.

**Which Git client (if any) did you choose? Why?**

I've been using the command line through PowerShell so far, since that's what I already had set up and it's been straightforward for the add, commit, push cycle I've needed for each issue. I'm going to start trying VS Code's built in Git panel (Source Control tab) alongside it though, mainly to see staged and unstaged changes visually rather than relying on `git status` every time, and to have a quicker way to review a diff before committing.

**What was the most interesting thing you learned about Git today?**

Honestly, the most useful realisation was just how much a GUI panel can simplify things that are easy to fumble on the command line, like seeing exactly what changed in a file before committing it, instead of trusting that `git add` picked up the right thing. Command line Git works fine once you know the commands, but having a visual view of changes feels like it'll cut down on small mistakes, like committing the wrong file or forgetting to check a diff first.
